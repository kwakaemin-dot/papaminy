## Phase 1: MVP 기반 구축 예산 및 일정 계획

이 계획은 **시스템 안정성($S_{LER/PL}$)을 최우선 전제 조건**으로 설정하고, P95 Latency $< 500ms$ 및 $99.9\%$ Uptime 목표를 달성하는 것을 핵심 성공 지표(KPI)로 삼습니다.

### 1. 핵심 산출물 및 범위 정의 (MVP Scope)

Phase 1의 MVP는 **"시스템 안정성을 증명하고 최소한의 수익화 흐름을 검증할 수 있는 핵심 백엔드 파이프라인 구축"**에 초점을 맞춥니다.

| 항목 | 목표 및 요구사항 | $S_{LER/PL}$ 연관성 |
| :--- | :--- | :--- |
| **데이터 파이프라인** | 외부 데이터 입력부터 최종 지표 출력까지의 비동기 흐름 구축 (Latency 최적화 포함). | 핵심 안정성 확보 ($\text{Retry}/\text{Failover}$) |
| **안정성 모니터링** | $S_{LER/PL}$ 지표($\text{Uptime}, P95 Latency$)를 실시간으로 측정하고 시각화하는 로직 구현. | 수익화 기반(신뢰 구축) |
| **Fallback Mechanism** | 비동기 실패 시 안전 모드(Safe Mode)로 즉시 전환되는 Failover/Retry 메커니즘 완벽 구현. | 시스템 연속성 보장 (최우선 요구사항) |
| **데이터 모델** | 최소 기능 제품에 필요한 핵심 데이터 스키마 정의 및 검증 로직 통합. | MVP의 최소 기능 충족 |

### 2. 예산 계획 (Cost Estimation)

예산은 인건비(내부/외부), 도구 사용료, 그리고 안정성 테스트 기간을 포함하여 산정합니다. (개발팀 리소스를 최대한 활용하는 것을 전제로 합니다.)

| 항목 | 세부 내용 | 예상 비용 (단위: Man-Month) | 비고 |
| :--- | :--- | :--- | :--- |
| **인건비 (핵심 개발)** | 백엔드 아키텍처 설계 및 $S_{LER/PL}$ 로직 구현 (코다리 역할) | 4 MM | 시스템 안정성 확보에 집중 투자 |
| **인건비 (프론트/통합)** | 데이터 시각화 대시보드(KPI 모니터링 UI) 통합 및 Fallback UI 개발 | 2 MM | 사용자 경험 및 신뢰 구축 |
| **도구/인프라** | 클라우드 환경 설정, 모니터링 툴 라이선스 (초기 3개월) | $500$ USD | 안정성 인프라 확보 비용 |
| **테스트 및 검증** | 부하 테스트(Latency/Uptime) 및 안전 모드 시나리오 테스트 | 1 MM | $S_{LER/PL}$ 목표 달성 보장 |
| **총 예상 예산 (Phase 1)** | - | **7 MM + $500$ USD** | 안정성을 위한 최소 투자 비용 |

### 3. 개발 일정 계획 (Timeline)

전체 프로젝트 기간은 시스템의 복잡성과 요구되는 안정성 검증 시간을 고려하여 총 **6주**로 설정합니다.

| 주차 | 주요 목표 (Milestone) | 핵심 활동 내용 | $S_{LER/PL}$ 중점 사항 |
| :--- | :--- | :--- | :--- |
| **Week 1** | **기반 설계 완료** | Fallback Mechanism 및 Latency 측정 로직의 최종 코드 구현 및 초기 데이터 모델 확정. | $T_{stability}$ 임계값 정의 및 기본 구조 확립. |
| **Week 2** | **핵심 파이프라인 구현** | 비동기 작업 및 외부 API 호출에 대한 Retry/Failover 로직의 기능적 통합 검증. | Failover 시나리오 테스트 (3회 재시도 포함). |
| **Week 3** | **안정성 측정 통합** | $S_{LER/PL}$ 지표를 실시간으로 기록하고 대시보드에 표시하는 모니터링 레이어 구현. | $P95 Latency$ 측정 로직의 정확성 검증. |
| **Week 4** | **MVP 기능 완성** | 최소한의 데이터 입력 및 결과 출력 흐름을 End-to-End로 연결하여 MVP 기능 완성. | 전체 시스템이 안정적인 상태에서 작동하는지 최종 검증. |
| **Week 5** | **부하 테스트 및 안정화** | 목표 $99.9\%$ Uptime 달성을 위한 부하 테스트 수행 및 병목 지점 최종 제거. | 시스템 안정성($S_{LER/PL}$)의 실시간 모니터링 집중. |
| **Week 6** | **최종 검토 및 배포 준비** | 모든 안전장치(Fallback)가 정상 작동함을 확인하고, 운영팀 인계 문서화 완료. | 최종 보고서 및 기술 문서 패키징. |
