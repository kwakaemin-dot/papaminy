### 📂 맛집 정보 파일 시스템 구조 (초안 설계)

모든 콘텐츠가 체계적으로 저장되고 검색이 용이하도록 **지역 기반 계층 구조**를 제안합니다. 이는 향후 데이터 분석 및 보고에 최적화된 구조입니다.

**기본 경로:** `~/맛집_데이터/`

| 레벨 | 폴더/파일 이름 | 목적 및 설명 |
| :--- | :--- | :--- |
| **1단계 (지역)** | `[Region_Name]/` | 모든 맛집 정보를 지역별로 완벽하게 분리하여 관리합니다. |
| **2단계 (카테고리)** | `01_기본정보/` | 각 식당의 핵심 데이터(주소, 연락처, 기본 정보)를 저장합니다. |
| | `02_콘텐츠/` | Tension $\rightarrow$ Release $\rightarrow$ Expectation 등 콘텐츠 원본을 저장합니다. (Writer 산출물) |
| | `03_분석/` | 데이터 기반 분석 결과(KPI, 트렌드 요약)를 보관합니다. |
| **최상위** | `[Region_Name]/` | 예: `~/맛집_데이터/Seoul/`, `~/맛집_데이터/Busan/` |

#### 💡 폴더 구조 상세 예시 (예시: 서울)

```
~/맛집_데이터/
├── Seoul/
│   ├── 01_기본정보/
│   │   ├── Restaurant_A.md  (식당 A의 기본 정보 파일)
│   │   └── ...
│   ├── 02_콘텐츠/
│   │   ├── Tension_Release_Expectation_Seoul_Intro.md (T-R-E 프레임워크 콘텐츠)
│   │   └── Review_Post_A.md      (실제 리뷰 및 경험 기록)
│   └── 03_분석/
│       └── Seoul_KPI_Analysis.md (지역별 수익화 잠재력 분석)
├── Busan/
│   └── ...
└── Gyeonggi/
    └── ...
```
