### 2. 데이터 흐름 구축을 위한 요구사항 목록 (Requirements for Data Flow Setup)

성공적인 수익화 측정 시스템(Data Flow)을 구축하기 위해 다음 세 가지 영역에 대한 요구사항을 명세합니다. 이는 향후 자동 측정 시스템 설계의 기반이 됩니다.

#### A. 인증 및 보안 요구사항 (Authentication & Security)
1.  **API Key 확보:** PayPal Developer Dashboard에서 **Client ID와 Client Secret**을 정확하게 발급받아야 합니다.
2.  **보안 저장소 정의:** 확보된 자격 증명은 시스템 내에서 안전하게 관리되어야 하며, 파일 시스템 접근 시 반드시 암호화 또는 환경 변수를 통해 주입되어야 합니다. (예: `paypal_revenue.json` 파일의 보안 강화)

#### B. 데이터 흐름 요구사항 (Data Flow Requirements)
1.  **입력 정의:** 수익화 모델 가설에 따라, 어떤 종류의 거래 데이터(매출, 결제 수단별 분류 등)를 측정할지 명확히 정의해야 합니다. (예: 상품별 매출, 환불율, 평균 거래 금액 등)
2.  **처리 로직 설계:** Raw 데이터가 들어왔을 때, 이를 '후킹 프레임워크'의 KPI(시청 지속률 극대화와 연관된 지표)로 변환하는 **변환 함수(Transformation Function)**의 논리적 구조를 설계해야 합니다.
3.  **출력 및 시각화 명세:** 최종적으로 도출될 KPI 대시보드에 필요한 데이터 필드(Metric Fields) 목록과 시각화 방식(차트 종류, 색상 코드 등)을 정의해야 합니다.

#### C. 시스템 통합 요구사항 (System Integration Requirements)
1.  **자동 실행 스케줄링:** 데이터 확보 및 분석 과정이 수동 개입 없이 주기적으로(예: 일별, 주별) 자동으로 실행되도록 Cron 또는 유사한 스케줄링 메커니즘을 설계해야 합니다.
2.  **오류 처리 로직:** API 연결 실패, 데이터 형식 오류 발생 시 시스템이 중단되지 않고, 에러 로그를 기록하며 다음 사이클로 넘어갈 수 있는 **강건한(Robust) 예외 처리 로직**을 구축해야 합니다.
