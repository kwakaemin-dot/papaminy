## 💎 핵심 비주얼 시스템: Emotional Flow System 적용

기존의 '긴장($\rightarrow$) 해소($\rightarrow$) 기대' 흐름에 맞춰 Black과 Gold의 사용 규칙을 구체화하여 MVP에 적용할 기반을 마련합니다.

*   **Chaos/Tension (불안정성 구간):** 깊은 블랙 배경 ($\#000000$) 위에 노이즈, 글리치 효과 또는 깨진 선(Glitch/Noise) 질감을 사용하여 혼란과 위험을 표현합니다.
*   **Control/Release (해결책 제시 구간):** 안정성과 통제는 명확하고 정돈된 **골드 톤($\#FFD700$ 또는 더 깊은 황금색)**의 선명한 라인, 깔끔한 그리드, 그리고 미니멀한 타이포그래피를 통해 표현합니다.
*   **Expectation (기대 구간):** 최종 결과물 및 안정성은 골드와 블랙의 완벽한 대비 속에서 명확하게 제시되어 신뢰감을 극대화합니다.

## ✨ 썸네일 기반 최종 비주얼 콘셉트 5가지 제안

영상 전체 흐름을 대표하며 CTR 극대화를 목표로 하는 5가지 핵심 시각 컨셉입니다.

### **Concept 1: The Glitch & Gold Reveal (Hook 집중)**
*   **목표:** 가장 강력한 후킹(문제 제기)에 초점. 즉각적인 혼란 유발.
*   **구성:** 배경 전체를 블랙 노이즈로 채우고, 중앙에 거대한 '불안정성' 데이터 그래프가 심하게 글리치(Glitch) 효과를 일으킵니다. 핵심 문구는 불안정성의 중심에 골드색으로 깜빡이며 배치됩니다.
*   **감정 흐름:** $\text{긴장 극대화}$ (Chaos).

### **Concept 2: The System Switch (Tension & Release)**
*   **목표:** 시스템 전환의 드라마틱한 순간을 시각화.
*   **구성:** 화면이 대각선으로 분할됩니다. 왼쪽(불안정)은 어둡고 거친 블랙/레드 계열, 오른쪽(안정)은 매끄럽고 광택 있는 골드 그라데이션 영역입니다. 중앙에는 '스위치' 또는 '게이트' 형태의 시각적 오브젝트가 전환되는 순간을 포착합니다.
*   **감정 흐름:** $\text{긴장} \rightarrow \text{해소}$ (Switch).

### **Concept 3: The Controlled Grid (Release & Control)**
*   **목표:** 시스템 안정성(Control)의 명확한 제시.
*   **구성:** 완벽하게 정렬된 골드 라인과 블랙 배경 위에서 데이터가 완벽히 제어되고 있음을 보여주는 미니멀하고 깔끔한 UI 형태를 사용합니다. 모든 요소는 그리드 시스템 내에 정확히 위치하며, 여백(Negative Space)을 적극 활용합니다.
*   **감정 흐름:** $\text{해소} \rightarrow \text{기대}$ (Control).

### **Concept 4: The Gold Flow (Aesthetic Focus)**
*   **목표:** 브랜드의 고급스러움과 가치를 강조.
*   **구성:** 블랙 배경에 골드 선과 도형을 사용하여 데이터 흐름(Flow) 자체를 예술적으로 표현합니다. 복잡한 데이터 대신, 핵심적인 안정성 수치들만 빛나는 골드 라인으로 연결하여 시각적 아름다움을 극대화합니다.
*   **감정 흐름:** $\text{기대}$ (Aspiration).

### **Concept 5: The Error State (Warning)**
*   **목표:** 잠재적인 위험을 경고하며 행동을 촉구.
*   **구성:** 데이터의 일부가 불안정하게 깨지거나 오류 메시지가 골드 색상으로 강조되어 나타나며, 이는 '지금 시스템을 점검해야 한다'는 긴급성을 전달합니다.
*   **감정 흐름:** $\text{긴장} \rightarrow$ (행동 유도).

## 📐 MVP UI/UX 프로토타입 초기 와이어프레임 실행 계획

위의 5가지 콘셉트를 기반으로, 사용자에게 가장 효과적으로 '안정성'을 전달할 수 있는 **Concept 3: The Controlled Grid**를 주력으로 하여 와이어프레임을 설계합니다.

### **1단계: 정보 구조 및 레이아웃 정의 (Wireframe Blueprint)**

| 화면 영역 | 시각적 목표 | 디자인 적용 원칙 | 핵심 요소 |
| :--- | :--- | :--- | :--- |
| **A. 대시보드 (Home Screen)** | 안정성과 통제감 즉시 전달 | Concept 3 기반의 정렬된 그리드 시스템 | 중앙에 현재 $S_M$ 스코어(Gold), 주변에 주요 지표를 블랙 배경으로 배치. |
| **B. 데이터 분석 (Detail View)** | 복잡한 데이터를 명확하게 해석 | 골드 라인을 활용한 차트 강조 | 캔들스틱 차트를 골드로 감싸고, 핵심 추세선은 두꺼운 골드 라인으로 표시. |
| **C. 설정/안정화 모듈 (Control Module)** | 사용자가 직접 조작한다는 느낌 부여 | 명확하고 직관적인 토글 및 슬라이더 | 'Stability Score' 슬라이더를 중심으로, 조정 시 Gold 색상이 역동적으로 변화하도록 설계. |

### **2단계: 디자인 실행 계획 (Execution Plan)**

1.  **색상 팔레트 확정:**
    *   Primary Background: $\text{#0A0A0A}$ (거의 블랙)
    *   Accent/Control: $\text{#FFD700}$ (골드)
    *   Secondary Text: $\text{#E0E0E0}$ (밝은 회색)

2.  **타이포그래피 적용:**
    *   헤드라인 및 핵심 수치에는 **Monospace 계열의 산세리프체(예: IBM Plex Mono 또는 비슷한 기하학적 폰트)**를 사용하여 시스템적인 느낌을 부여합니다. (Black/Gold 대비 강조)

3.  **와이어프레임 프로토타입 제작:**
    *   Figma 환경에서 위 레이아웃 정의에 따라 **Concept 3**의 그리드 기반으로 초기 와이어프레임을 즉시 구축합니다. 모든 UI 요소는 골드 라인(Border, Highlight)을 통해 '제어되고 있음'을 시각적으로 암시해야 합니다.

4.  **반복 및 검증:**
    *   와이어프레임 완성 후, 레오 에이전트가 제안한 **Master Design System** 가이드라인에 따라 모든 UI 컴포넌트의 최종 스타일(Color Token 적용)을 적용하여 시각적 일관성을 최종 검증합니다.
